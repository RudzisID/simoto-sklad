# 001. Реализация навигации вкладок

**Status**: proposed

**Context**: frontend (планируется) | **Module**: public/app.js → будущая структура tabs/

**Related Tasks**: V3 — Новые фичи (PLAN.md)

## Context

Текущий фронтенд SiMOTO-sklad реализован как одностраничное приложение на Vanilla JS с единственной страницей `index.html`. По мере расширения функциональности (интеграция с WB/Ozon, печать этикеток, управление токенами) возникает потребность в структурировании UI через вкладки (tabs).

Проблемы текущего решения:
- Все функции доступны на одной странице, что ведет к разрастанию DOM и усложнению навигации
- Отсутствует возможность глубоких ссылок (deep linking) на конкретные разделы
- Смешение логики МойСклад, WB, Ozon в одном файле `app.js` (уже 2000+ строк)
- Нет изоляции состояния между разделами

## Decision

Принято решение реализовать навигацию вкладок со следующими характеристиками:

1. **Клиентский роутинг**: использовать hash-based роутинг (`#tab-moysklad`, `#tab-wb`, `#tab-ozon`) без сторонних библиотек
2. **Динамическая загрузка**: компоненты вкладок загружаются через dynamic import (ES modules) при первом обращении
3. **Управление состоянием**: активная вкладка определяется по `location.hash`, состояние каждой вкладки изолировано

### Структура вкладок:
- **МойСклад** (`#tab-moysklad`) — существующий функционал работы с заказами
- **Wildberries** (`#tab-wb`) — синхронизация и управление товарами WB
- **Ozon** (`#tab-ozon`) — синхронизация и управление товарами Ozon
- **Настройки** (`#tab-settings`) — управление токенами и конфигурацией

## Alternatives Considered

### Option 1: SPA фреймворк (React/Vue)
- **Pros**: Готовые решения для роутинга, состояния, компонентов
- **Cons**: Избыточно для текущего проекта, требует переписывания всего фронтенда, увеличивает размер бандла
- **Why rejected**: Проект имеет рабочий Vanilla JS код, полная миграция неоправданна на данном этапе

### Option 2: Multi-page приложение (отдельные HTML файлы)
- **Pros**: Полная изоляция, простота реализации
- **Cons**: Потеря состояния при переходах, дублирование общего кода (заголовок, панель вкладок), невозможность SSE между страницами
- **Why rejected**: SSE соединения и состояние заказов должны сохраняться при переключении вкладок

### Option 3: Iframe-based вкладки
- **Pros**: Полная изоляция кода и стилей
- **Cons**: Сложность коммуникации между iframe и родителем, проблемы с SSE, плохой UX
- **Why rejected**: Избыточная сложность для внутреннего инструмента

### Option 4: Vanilla JS с hash-роутингом (Выбрано)
- **Pros**: Минимальные изменения существующего кода, нет внешних зависимостей, сохранение SSE соединений
- **Cons**: Ручная реализация роутинга и загрузки компонентов
- **Why chosen**: Баланс между функциональностью и простотой внедрения

## Consequences

### Positive
- Структурирование UI без переписывания существующего кода
- Возможность глубоких ссылок (можно отправить ссылку на конкретную вкладку)
- Изоляция состояния между разделами (токены, данные)
- Уменьшение времени первичной загрузки за счет lazy loading
- Сохранение SSE соединений при переключении вкладок

### Negative
- Ручная реализация роутинга (нужно тестировать обработку hash change)
- Усложнение структуры фронтенда (нужно разделить `app.js` на модули)
- Необходимость реализации динамической загрузки компонентов

## Implementation Notes

### Hash-роутинг:
```javascript
// Обработка навигации
function handleRouteChange() {
  const hash = location.hash || '#tab-moysklad';
  const tabName = hash.replace('#tab-', '');
  activateTab(tabName);
}

window.addEventListener('hashchange', handleRouteChange);
window.addEventListener('DOMContentLoaded', handleRouteChange);
```

### Динамическая загрузка:
```javascript
// tabs/moysklad.js, tabs/wb.js, tabs/ozon.js, tabs/settings.js
export async function initTab(container) {
  // Инициализация вкладки
}

async function loadTab(tabName) {
  const module = await import(`./tabs/${tabName}.js`);
  return module;
}
```

### Структура файлов (планируется):
```
public/
├── index.html
├── app.js           # Корневая логика, роутинг
├── styles.css
└── tabs/
    ├── moysklad.js  # Логика текущего app.js (выделенная)
    ├── wb.js        # Wildberries интеграция
    ├── ozon.js      # Ozon интеграция
    └── settings.js  # Настройки и токены
```
