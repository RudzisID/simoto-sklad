# Auto-Push: Автоматический пуш на GitHub

**Файл**: `scripts/auto-push.js`

Скрипт для автоматического коммита, пуша и создания GitHub Release с версионированием.

## Использование

```bash
# Patch версия (1.0.0 → 1.0.1) — по умолчанию
node scripts/auto-push.js

# Minor версия (1.0.0 → 1.1.0)
node scripts/auto-push.js minor

# Major версия (1.0.0 → 2.0.0)
node scripts/auto-push.js major

# Автоматическое определение типа версии
node scripts/auto-push.js auto

# Тестовый прогон без пуша
node scripts/auto-push.js --dry-run
```

## Возможности

- **Версионирование** — автоматическое обновление версии в `package.json`
- **Анализ изменений** — определяет тип версии на основе измененных файлов:
  - `major` — изменения в `server.js`, `lib/` (API, moysklad)
  - `minor` — новые функции, изменения в `lib/`, `public/`
  - `patch` — по умолчанию
- **Тесты** — запускает `npm test` перед коммитом
- **Синхронизация** — делает `git pull --rebase` перед пушем
- **GitHub Release** — создает релиз через API (требует `GH_TOKEN`)
- **Откат при ошибках** — восстанавливает версию и коммит при неудаче

## Требования

1. **GitHub токен** в `.env`:
   ```env
   GH_TOKEN=ghp_xxxxxxxxxxxx
   ```
   Требуемые права: `repo`, `write:packages`

2. **Git настроен**:
   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "you@example.com"
   ```

3. **Доступ к репозиторию** `RudzisID/simoto-sklad`

## Процесс работы

1. Проверка GitHub токена
2. Проверка git статуса
3. Анализ изменений (файлы, diff)
4. Определение типа версии (или использование указанного)
5. Обновление версии в `package.json`
6. Запуск тестов (`npm test`)
7. Создание коммита
8. Синхронизация с `origin` (`git pull --rebase`)
9. Пуш на GitHub
10. Создание тега и GitHub Release

## Опции

| Аргумент | Описание |
|----------|----------|
| `patch` | Исправления (по умолчанию) |
| `minor` | Новые функции |
| `major` | Критические изменения |
| `auto` | Автоопределение типа |
| `--dry-run`, `-n` | Тестовый прогон без пуша |
| `--auto`, `-y` | Автоматический режим без подтверждения |
| `--help`, `-h` | Справка |
