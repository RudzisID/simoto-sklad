# Руководство по поддержке документации

Документация — это часть кодa. Актуальная документация помогает быстрее понимать код и избегать ошибок.

## Принципы документирования

### Когда документировать

1. **При серьезных изменениях** — рефакторинг, новые модули, изменение API
2. **При добавлении новых функций** — каждая публичная функция должна быть задокументирована
3. **При изменении существующих функций** — обновлять документацию вместе с кодом
4. **Перед релизом** — проверять актуальность всех `.md` файлов

### Синхронизация кода и документации

**Правило**: Документация должна обновляться в том же коммите, что и код.

```bash
# Плохо: код изменен, документация нет
git add server.js
git commit -m "Added new endpoint"

# Хорошо: код и документация вместе
git add server.js docs/API.md
git commit -m "Added new endpoint - updated docs"
```

## Формат документации

### Для функций

Каждая публичная функция должна иметь:

```javascript
/**
 * Краткое описание функции
 * @param {string} param1 - Описание параметра
 * @param {number} param2 - Описание параметра
 * @returns {Promise<object>} - Описание возвращаемого значения
 */
async function exampleFunction(param1, param2) {
  // ...
}
```

### Для модулей (lib/)

Файл `docs/lib/имя_модуля.md`:

```markdown
# Название модуля

**Файл**: `lib/module.js`

Краткое описание модуля (1-2 предложения).

## Экспортируемые функции

| Функция | Описание |
|---------|----------|
| `functionName` | Краткое описание |

## Примеры использования

\```javascript
const { functionName } = require('./lib/module')
const result = await functionName('param')
\```
```

### Для API эндпойнтов

В `docs/API.md` или отдельном файле:

```markdown
#### `POST /api/endpoint`

Описание эндпойнта.

**Запрос:**
\```json
{
  "param": "value"
}
\```

**Ответ:**
\```json
{
  "success": true
}
\```
```

## Автоматическая генерация

Используйте скрипт для обновления документации модулей:

```bash
npm run docs
```

Скрипт (`scripts/docs-generator.js`):
- Сканирует файлы в `lib/`
- Извлекает JSDoc комментарии
- Обновляет файлы в `docs/lib/`

## Чек-лист перед коммитом

- [ ] Все новые функции имеют JSDoc комментарии
- [ ] Обновлена документация в `docs/lib/` (если менялись модули)
- [ ] Обновлен `docs/API.md` (если менялись эндпойнты)
- [ ] Обновлен `README.md` (если менялась структура проекта)
- [ ] Запущен `npm run docs` для обновления автогенерируемых файлов
- [ ] Документация написана на русском языке

## Советы

1. **Примеры** — лучше 10 строк кода, чем 100 слов текста
2. **Краткость** — если описание занимает больше 30 секунд на чтение, упростите
3. **Актуальность** — не оставляйте устаревшие примеры, они хуже их отсутствия
4. **Согласованность** — используйте тот же стиль, что и в существующей документации
