# Wildberries Seller API - Лимиты запросов (Rate Limits)

## Алгоритм ограничения
Wildberries API использует алгоритм **token bucket** для равномерного распределения нагрузки.

## Общие лимиты (пример для Marketplace категории)

| Тип токена | Период | Лимит | Интервал | Burst (пачка) |
|------------|--------|-------|----------|---------------|
| Personal | 1 минута | 300 запросов | 200 мс | 20 запросов |
| Service | 1 минута | 300 запросов | 200 мс | 20 запросов |
| Base | 1 минута | 150 запросов | 200 мс | 10 запросов |
| Test | 1 минута | 150 запросов | 200 мс | 10 запросов |

### Параметры:
- **Period** — временной интервал, за который можно отправить максимальное количество запросов
- **Limit** — максимальное количество запросов за период (например, 300 в минуту)
- **Interval** — промежуток времени для пауз между запросами (например, 200 мс или 0.2 секунды)
- **Burst** — максимальное количество запросов, которые можно отправить одновременно без пауз

### Расчет интервала:
```
Interval = Period / Limit
Например: 60 секунд / 300 запросов = 0.2 секунды (200 мс)
```

## Заголовки ответов (Rate Limit Headers)

При успешных запросах (кроме 429 ошибки) возвращаются заголовки:

- **`X-Ratelimit-Remaining`** — количество запросов, которые можно отправить прямо сейчас без пауз
  - Уменьшается на 1 после каждого запроса
  - Если равен 0 — следующий запрос без задержки вернет ошибку 429
  - Восстанавливается со временем

- **`X-Ratelimit-Retry`** (только при 429) — количество секунд, через которые можно повторить запрос
  - Если попытаться раньше — продолжите получать 429

- **`X-Ratelimit-Limit`** (только при 429) — максимально допустимый burst запросов
  - Восстановится после X-Ratelimit-Reset секунд

- **`X-Ratelimit-Reset`** (только при 429) — количество секунд до восстановления лимита запросов

## Пример ответа 429 (Too Many Requests):
```http
HTTP/1.1 429 Too Many Requests
...
X-Ratelimit-Reset: 29
X-Ratelimit-Retry: 2
X-Ratelimit-Limit: 10
```

## Особые случаи учета запросов
Некоторые запросы считаются как несколько:
- **Ошибка 409** в категории Marketplace считается как **5 запросов** (вместо 1)
- Значение `X-Ratelimit-Remaining` уменьшается сразу на 5 единиц

## Лимиты для конкретных методов

### Ping методы (проверка соединения)
- **Лимит**: 3 запроса каждые 30 секунд
- Применяется индивидуально к каждому экземпляру метода на каждом хосте
- При программном использовании метод временно блокируется

### Content API (Контент)
Для всех методов в категории Content:
- **Лимит**: 100 запросов в минуту
- **Burst**: 5 запросов
- **Исключения** (другие лимиты):
  - Создание карточек товаров
  - Создание карточек с объединением
  - Редактирование карточек товаров
  - Получение ошибочных карточек
  - Перемещение в корзину/восстановление

### Feedbacks API (Отзывы и вопросы)
- **Лимит**: 3 запроса
- **Interval**: 333 мс
- **Burst**: 6 запросов

### Reports API (Отчеты)
Большинство методов создания отчетов:
- **Лимит**: 1 запрос в минуту
- **Burst**: 1-10 запросов (зависит от типа отчета)
- Некоторые отчеты: 1 запрос в 5 секунд или 10 секунд

### WBD API (Wildberries Digital)
- Создание/редактирование офферов: **50 запросов в секунду**
- Получение информации об оффере: **100 запросов в секунду**
- Список офферов с фильтрацией: **100 запросов в секунду**
- Загрузка контента: **10 запросов в секунду**
- Получение списка контента: **100 запросов в секунду**

## Рекомендации по работе с лимитами
1. Используйте `X-Ratelimit-Remaining` для определения возможности отправки запросов
2. При получении 429 — дождитесь `X-Ratelimit-Retry` секунд перед повтором
3. Распределяйте запросы равномерно согласно `Interval`
4. Не превышайте `Burst` без пауз
5. Учитывайте, что некоторые ошибки (409) считаются как несколько запросов
6. Для массовых операций используйте пакетные методы (batch endpoints)

## Обработка ошибки 429
```javascript
// Пример обработки на JavaScript
if (response.status === 429) {
    const retryAfter = response.headers.get('X-Ratelimit-Retry');
    console.log(`Превышен лимит. Повторите через ${retryAfter} секунд`);
    await sleep(retryAfter * 1000);
    // Повторить запрос
}
```
