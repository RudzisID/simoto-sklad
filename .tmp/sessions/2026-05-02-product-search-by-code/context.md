# Исправление поиска товара по коду (code) для печати стикеров

## Проблема

При попытке печати стикера товара с кодом, содержащим кириллицу (например `0180-051003-0003комплект`), поиск не находит товар:

```
Печать стикера: 0180-051003-0003комплект
Поиск товара по коду: 0180-051003-0003комплект
Searching product with: search=0180-051003-0003%D0%BA%D0%BE%D0%BC%D0%BF%D0%BB%D0%B5%D0%BA%D1%82
Search failed, trying exact filter for: 0180-051003-0003комплект
Product not found: 0180-051003-0003комплект
Товар не найден: 0180-051003-0003комплект
```

## Анализ

### 1. Документация МойСклад API

Из официальной документации МойСклад JSON API 1.2:

> **"Значение этого параметра - urlencoded строка с поисковыми условиями, перечисленными через `;`"**

МойСклад API поддерживает:
- Кириллицу в полях `name`, `code`, `article`
- Контекстный поиск через параметр `search` (ищет по name, code, article одновременно)
- Фильтрацию через `filter=code=значение`

### 2. Библиотека `moysklad` (v0.21.1)

Исходный код библиотеки показывает:

**`parseQueryString.js:17`** — декодирует query-параметры:
```js
return decodeURIComponent(str)
```

**`buildQuery.js:17`** — кодирует значения параметров:
```js
res.push([key, encodeURIComponent(val)])
```

**Путь URL при вызове `API.GET('entity/product?search=...')`:**
1. `parseUrl` разбирает URL, декодирует query-параметры через `decodeURIComponent`
2. `buildQuery` кодирует значения обратно через `encodeURIComponent`
3. Итоговый URL отправляется на сервер

### 3. Текущий код (`lib/product.js`)

**Строка 31:**
```js
const searchQuery = `search=${encodeURIComponent(normalizedCode)}`
```

**Строка 34:**
```js
const result = await API.GET('entity/product?' + searchQuery)
```

**Проблема:** Мы вызываем `encodeURIComponent` заранее, а затем библиотека `moysklad` декодирует и снова кодирует значение. Это приводит к корректному кодированию, но...

### 4. Корневая проблема

**Параметр `search` в МойСклад API ищет по name, code, article одновременно**, но возвращает результаты в порядке релевантности. Если товар с кодом `0180-051003-0003комплект` не находится, значит:

1. **`search` нашёл другие товары**, где `0180-051003-0003` встречается в name или article, но не нашёл точного совпадения по code
2. **Точный фильтр `filter=code=...` не нашёл товар**, потому что поле `code` в МойСклад может не содержать кириллицу (хотя пользователь утверждает, что содержит)

### 5. Решение

**Вариант A (рекомендуемый):** Использовать правильный API-вызов через объект `query`, а не через строку URL:

```js
// Вместо:
API.GET('entity/product?search=' + encodeURIComponent(code))

// Использовать:
API.GET('entity/product', { search: code })
```

Библиотека `moysklad` сама закодирует значение правильно.

**Вариант B:** Добавить fallback на поиск по `article` если `code` не нашёл.

**Вариант C:** Добавить логирование количества результатов search для диагностики.
